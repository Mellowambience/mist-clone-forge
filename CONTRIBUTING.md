# Contributing

1. Keep clones **narrow** — one intent per vessel.  
2. Prefer **absorb** over silent deletion.  
3. Lessons must come from **real acts** (local/retrieved), not vibes.  
4. Do not commit live `~/.grok/mist-clones` operator data.  
5. PRs: small, reversible, with a lesson line if behavior changes.

## Suggest a genesis specialist

Open an issue with:

- `id` (mist-*)  
- one-sentence intent  
- domain  
- ≤6 skill routes  
- why it is not already covered by an existing clone
